## 기존 서비스의 불편한 점

## 1️⃣ 추천 시스템의 한계

### ❌ “맞긴 한데, 왜 추천됐는지는 모르겠다”

### 현재 상태

- 대부분의 OTT 추천은
    
    **시청 이력 + 유사 사용자 + 장르 태그** 기반
    
- 결과는 맞을 때도 있지만 **설명이 없음**

### 구조적 한계

- 추천 모델이 **블랙박스**
- 사용자는:
    - “이거 왜 추천됐지?”
    - “난 이런 건 싫어하는데?”

👉 **추천의 ‘이유’를 제공하지 않음**

### 프로젝트로 풀 수 있는 포인트

- Explainable Recommendation
- “당신이 좋아한 ○○영화의 ○○요소 때문에 추천”
- LLM + 벡터 유사도 설명

## 2️⃣ 평점·리뷰의 신뢰성 문제

### ❌ “평점은 높은데 내 취향은 아니더라”

### 현재 상태

- IMDb / Rotten / 네이버 평점은 **대중 평균**
- 리뷰는 텍스트로 흩어져 있음

### 구조적 한계

- **개인 취향 반영 불가**
- 리뷰가 많을수록 오히려 판단 어려움
- 스포일러 섞여 있음

👉 “이 영화가 좋은가?”는 알 수 있어도

👉 “**나한테** 좋은가?”는 알기 어려움

### 프로젝트 포인트

- 리뷰 요약 + 감정 분석
- “당신과 취향이 비슷한 사람 기준 평점”
- 스포일러 제거 요약

## 3️⃣ 콘텐츠 과잉 (Choice Overload)

### ❌ “볼 게 너무 많아서 안 보게 된다”

### 현재 상태

- OTT 하나에 수천 개 콘텐츠
- 홈 화면 스크롤만 하다 종료

### 구조적 한계

- 추천은 많지만 **결정 지원은 없음**
- 사용자의 현재 상황(시간, 기분)을 고려하지 않음

👉 선택의 문제를 해결하지 못함

### 프로젝트 포인트

- 상황 기반 추천
    - “지금 30분 남았을 때”
    - “가볍게 볼 수 있는”
- 결정형 추천 (Top 3 + 이유)

## 4️⃣ 콘텐츠 이해 보조 기능 부족

### ❌ “이 장면 이해 못 했는데 되돌리기 귀찮다”

### 현재 상태

- 자막은 있지만 **이해 보조는 없음**
- 복선, 인물 관계, 세계관 설명 없음

### 구조적 한계

- OTT는 **‘재생 플랫폼’**에 집중
- 콘텐츠 해석은 사용자 몫

### 프로젝트 포인트

- AI 동반 시청
- “지금 장면 요약”
- “이 인물 누구야?”
- “스포일러 없이 설명”

👉 **OTT를 ‘혼자 보는 서비스’에서 ‘도와주는 서비스’로**

## 5️⃣ 함께 보기 기능의 한계

### ❌ “같이 보긴 하는데, 대화가 불편하다”

### 현재 상태

- Watch Party / Discord
- 단순 채팅 위주

### 구조적 한계

- 대화 흐름 정리 ❌
- 의견 요약 ❌
- 감상 기록 ❌

### 프로젝트 포인트

- AI가 대화 요약
- 장면별 반응 분석
- “우리의 감상 노트 자동 생성”

## 6️⃣ OTT는 데이터를 쌓지만, 사용자는 얻는 게 적다

### ❌ “내 취향이 정리되어 있지 않다”

### 현재 상태

- OTT는 시청 데이터를 **독점**
- 사용자는 자기 취향을 **이식/이동 불가**

### 구조적 한계

- 플랫폼 락인
- 사용자 취향의 구조화 부재

### 프로젝트 포인트

- 개인 취향 프로필 생성
- “나의 영화 취향 리포트”
- OTT 간 추천 공용화 (개념적으로)

# 서비스 공통 기능

## [Chatgpt]

### A. 평가·리뷰

- **별점/평점 시스템**(0.5 단위, 0.1 단위, 1~10 등)
- **텍스트 리뷰/코멘트 작성**, 리뷰 열람
- 리뷰 **좋아요/반응**, 댓글/대댓글(일부)
    - 근거 예: 별점·리뷰 작성, 텍스트 리뷰, 코멘트, 좋아요/반응 영화관련 서비스별 기능

### B. 탐색·검색

- **작품 검색**, DB 조회, 상세 정보(장르/캐스트/줄거리 등) → IMDb API 불러오는듯?
- **필터/정렬**, 해시태그/태그 검색(일부)
    - 근거 예: 작품 검색·상세 정보, 리뷰 검색·필터링, 키워드/장르/검색 필터

### C. 개인화·기록

- **시청 기록/로그**, 진행률 추적(에피소드 단위 포함)
- **관심 목록(Watchlist)** / 보고 싶은 목록
- **개인 통계/랭킹/대시보드**
    - 근거 예: 개인 시청 기록 관리, 시청 기록 및 진행률 추적, watchlist, 연간 통계

### D. 소셜·커뮤니티

- **사용자 프로필**, 팔로우, 활동 피드
- 커뮤니티 토론/게시판/포럼(일부), SNS 공유
- 알림(새 리뷰/댓글/팔로우 새 글 등)
    - 근거 예: 사용자 프로필·팔로우, 커뮤니티 토론/게시판, 알림, SNS 공유

### E. 메타 정보·부가 콘텐츠

- **트레일러/예고편**
- **랭킹/차트**, 박스오피스(일부)

# Chatgpt - 기존 기능의 한계와 문제점 / LLM, RAG 활용 방안

# 1) 기존 기능의 한계와 부족한 점

## 문제 ① 취향 모델링이 “얕다”

- 대부분은
    - 장르
    - 시청 이력
    - 단순 평점
        
        정도만 사용
        
- 사용자의 **정서적 취향(어두운 톤, 인간관계 중심, 철학적 메시지 선호 등)**을 모델링하지 못함
- 결과: 추천이 반복되고 금방 식상해짐

👉 *문제 본질:* 취향을 “장르 벡터” 수준으로만 이해함

---

## [해결]

## ① 정서 기반 취향 모델링

### 기능

LLM이 사용자의 리뷰·코멘트·대화 기록을 분석해

- 선호 감정 톤
- 서사 구조 선호
- 캐릭터 유형 선호
- 결말 선호

를 추출

### 차별점

“액션 좋아함”이 아니라

“성장형 서사 + 여운 있는 결말 선호” 같은 수준

### GPT의 구현가능성

정리하면, **영화·OTT 추천 분야에서 “LLM으로 사용자 리뷰/코멘트/대화 같은 비정형 텍스트를 분석하는 연구는 최근 각광받고 있지만 아직 초기 단계이거나 제한적으로만 활용되고 있는 게 사실이야.** 아래 근거를 정리했어 👇

---

## ✅ 1) LLM을 완전한 추천 엔진으로 쓰는 연구는 아직 초기 단계

- 일부 연구는 **LLM 자체를 추천 생성에 활용**하는 가능성을 보였지만 아직 **일반적인 추천 모델처럼 널리 쓰이지는 않음**이라고 평가돼.
    
    → 예: LLM이 자연어 사용자 요청을 받아 **대화형 추천**을 해보는 시도 연구가 있으나,
    
    **일반적인 개인화·정확도 측면에서는 한계가 있다는 의견도 존재함.**
    

---

## ✅ 2) LLM을 기존 추천 시스템 보조/설명 생성 용도로 활용하는 연구는 존재함

- LLM은 **설명 가능성(explainability) 제공 분야에서 사용됨** → “왜 이 영화를 추천했는지” 같은 자연어 설명 생성.
    
    → 그러나 **추천 성능 향상 연구는 아직 많지 않음**(논문이 소수).
    

---

## ✅ 3) “리뷰 텍스트 자체를 감성분석/추천에 활용”하는 연구는 존재함 (LLM까지는 아님)

- 일부 최신 영화 추천 연구는 **사용자 리뷰 텍스트를 감성분석해 추천 정확도를 높이는 시도**를 보고함
    
    → 리뷰 기반 추천 정확도가 전통적 평점 기반보다 향상.
    
    → 과거 연구는 BERT 같은 언어모델을 활용했지만, **LLM (GPT-계열) 연구는 아직 대체적이지 않음.**
    

---

## 🚫 즉, 요즘 영화 추천 시스템들은…

### ❌ 대부분은 아직도

- 평점 기반 추천
- 협업 필터링/콘텐츠 기반 벡터 추천
- 일부 감성분석 기반 하이브리드 추천
    
    이 중심이야
    

### ❗ 그리고 지금까지의 LLM 활용은

✔ LLM **설명 생성/대화형 추천 연구는 있음**

✔ **LLM으로 리뷰/대화 텍스트를 취향 모델링까지 완전히 대체해 쓰는 상용 시스템은 드묾**

→ 즉 여러분이 제안한

> “LLM으로 리뷰·대화·코멘트를 분석해서 정서/서사/캐릭터까지 취향 프로파일로 만든다”
> 
> 
> 이건 **현업 상용 서비스에는 아직 거의 도입되지 않았거나 연구 중인 최첨단 영역**이라고 할 수 있어.
> 

---

## 📌 왜 아직 널리 쓰이지 않을까?

### ❶ 기존 추천은 구조화 데이터 중심

- 평점 / 시청이력 같은 **정형 데이터**가 주로 추천 모델 입력임

### ❷ 텍스트 분석은 복잡

- 리뷰·코멘트는 단일 점수보다 **맥락/스포일러/토픽** 같은 **심층 정보**가 필요함
- LLM은 이런 **정서/서사 분석**에 강하지만,
    
    ➤ 추천 정확도 향상/평가가 어렵고
    
    ➤ 학습/비용 부담이 큼
    
    ➤ 사용자 벡터로 정량화하는 부분이 아직 연구 중임
    

---

## 🎯 정리: 너의 해결 방안은 “최신 연구 트렌드 + 아직 상용화 안 된 혁신 영역”

✔ LLM으로 리뷰/대화/코멘트 텍스트를 심층 분석 →

정서/서사/캐릭터 기반 취향 표현 만들기

→ **기존 추천 시스템에서는 거의 안 하고 있는 접근**임

✔ 논문 리뷰에서도

- 일부는 **LLM을 추천 설명・대화형 상호작용에 사용**함
- 리뷰 텍스트 기반 추천 연구는 있으나 **LLM 전체를 추천 엔진 입력으로 활용한 논문은 많지 않음**

---

## 💡 따라서 너의 주장에 대한 근거

### 👌 “현재 영화 추천은 리뷰/코멘트 기반 정서적 취향 모델링을 잘 못한다”

→ 맞음. 대부분 기존 추천은 **정형 데이터 + 평점 중심**인 것이 사실임.

### 👌 “LLM을 써서 정서·서사·캐릭터 선호까지 반영한다면 차별화된다”

→ 맞고, **현재 연구도 이 방향을 모색하는 중**임.

### 👀 “상용 OTT에서 이걸 쓰고 있다”라고 말하기는 아직 힘듦

→ 아직 널리 사용되지 않았고, 대체로 **연구/프로토타입** 수준이라서 그 점은 고려해야 함.

---

📌 “실제 구현 가능한 데이터 파이프라인 + LLM 프롬프트 구조”

# 1️⃣ 자연어 감성 검색 (우선순위 ★★★★★)

## ✅ 왜 현실적?

- RAG 기본 구조만 있으면 가능
- 대규모 개인화 모델 필요 없음
- DB만 잘 구성하면 됨

## ✅ 기능 예시

사용자 입력:

> “비 오는 날 혼자 보기 좋은 잔잔한 영화”
> 

LLM 역할:

- 감성 키워드 추출
    
    → 잔잔함, 감성, 고독, 힐링
    

RAG 역할:

- 영화 메타데이터 + 줄거리에서 매칭

## ✅ 구현 구조

- TMDB/OMDb API로 영화 데이터 수집
- 줄거리 임베딩 후 벡터DB 저장
- 사용자 질의 임베딩 → 유사 검색

## ✅ 데모 임팩트

매우 큼

→ 심사자들이 바로 체감함

---

# 2️⃣ AI 리뷰 요약 + 스포일러 제거 (우선순위 ★★★★★)

## ✅ 왜 현실적?

- 개인화 없이도 가능
- LLM 단독으로 구현 가능
- 데이터만 모으면 됨

## ✅ 기능

한 영화의 리뷰 100개 →

LLM이:

- 핵심 장단점 요약
- 스포일러 문장 제거
- 전체 감정 톤 분석

## ✅ 구현

- 크롤링 or 공개 리뷰 데이터

# Claude - 기존 기능의 한계와 문제점 / LLM, RAG 활용 방안

## 1. 기존 기능의 한계와 부족한 점

### **평가 시스템의 한계**

- **단순 별점의 문제**: 0.5~1점 단위 별점으로는 미묘한 감정 차이 표현 불가
- **평가 기준 불명확**: "왜 3.5점인지" 본인도 설명하기 어려움
- **시간에 따른 평가 변화 미반영**: 처음 본 순간과 시간이 지난 후의 평가가 다를 수 있음

### **리뷰 작성 및 활용의 문제**

- **리뷰 작성 피로도**: 매번 긴 텍스트 작성은 부담
- **리뷰 검색의 어려움**: 원하는 관점의 리뷰 찾기 어려움
- **스포일러 회피 불완전**: 스포일러 태그만으로는 부족
- **리뷰 품질 편차**: 유용한 리뷰와 감정적 리뷰 구분 어려움

### **추천 시스템의 한계**

- **협업 필터링의 콜드스타트**: 신규 사용자는 충분한 데이터 없음
- **장르 기반 추천의 단순함**: 같은 장르여도 분위기/스타일 천차만별
- **비슷한 작품만 추천**: 필터 버블에 갇혀 새로운 발견 어려움
- **OTT별 추천 편향**: 자사 콘텐츠 위주 추천 가능성

### **시청 기록 관리의 한계**

- **단순 목록 나열**: "언제 봤는지"만 기록
- **시청 맥락 부재**: "왜 봤는지", "누구와 봤는지" 기록 없음
- **감상 변화 추적 불가**: 재시청 시 느낌의 변화 기록 안됨

### **검색 및 필터링의 문제**

- **정확한 조건을 알아야 함**: "따뜻한 느낌", "몽환적인" 같은 감성 검색 불가
- **복합 조건 검색 제한**: "코미디인데 슬픈 장면도 있는" 같은 세밀한 검색 어려움
- **장면/대사 기반 검색 불가**: "그 대사가 나오는 영화" 찾기 어려움

### **소셜 기능의 한계**

- **팔로우 시스템의 피상성**: 취향 유사도를 모르고 팔로우
- **댓글의 산발성**: 체계적인 토론 구조 없음
- **취향 그룹 형성 어려움**: 비슷한 취향의 사람들 자동 연결 안됨

### **콘텐츠 정보의 한계**

- **정형화된 정보만 제공**: 장르, 출연진, 줄거리 정도
- **분위기/톤 정보 부족**: 실제 느낌을 전달하는 메타데이터 없음
- **관계성 정보 부족**: "이 영화와 저 영화의 관계" 설명 없음

### **에피소드 관리 (드라마/시리즈)**

- **에피소드별 품질 편차 정보 없음**: "3화는 지루한데 4화부터 재밌음" 같은 정보 부재
- **중간 진입 어려움**: 시즌 중간부터 보기 시작할 때 맥락 파악 어려움

## 2. LLM/RAG로 추가할 수 있는 차별화 기능

### **🎯 감성 기반 검색 (Emotional Search)**

`사용자: "비 오는 날 보면 좋을 만한 영화"
시스템: 시각적 분위기(흐린 날씨, 실내 장면), 음악(잔잔한 재즈), 
       감정 톤(고독, 사색) 분석하여 추천`

- LLM이 자연어 쿼리를 감성 벡터로 변환
- RAG로 영화의 시각/청각적 특징, 리뷰의 감정 분석 데이터 검색
- "몽환적인", "따뜻한", "긴장감 넘치는" 같은 추상적 검색 지원

### **📝 자동 마이크로 리뷰 생성**

`사용자가 별점만 입력하면:
"연출과 음악은 훌륭했지만, 후반부 전개가 다소 급했습니다. 
 배우들의 연기는 인상적이었어요."`

- 사용자의 과거 리뷰 패턴 학습
- 별점 + 간단한 키워드만으로 LLM이 리뷰 초안 생성
- 사용자가 수정하거나 그대로 게시 선택

### **🔍 "이런 장면" 찾기 기능**

`"주인공이 비 맞으면서 우는 장면 있는 영화"
"반전이 충격적인데 복선이 잘 깔린 스릴러"
"요리하는 장면이 자주 나오는 영화"`

- RAG로 영화 스크립트, 장면 설명, 리뷰에서 특정 요소 검색
- 시각적/서사적 요소 기반 검색 가능

### **👥 취향 DNA 매칭 (Taste DNA Matching)**

`"당신과 87% 취향이 일치하는 사용자 @cinephile23이 
 최근 높게 평가한 작품입니다"`

- 사용자별 취향 벡터 생성 (시청 이력, 평점, 리뷰 분석)
- 유사도 높은 사용자 자동 연결
- LLM이 "왜 비슷한지" 설명 생성 ("두 분 모두 느린 템포의 드라마를 선호하십니다")

### **📊 개인화 평점 예측 및 설명**

`"이 영화는 일반 평점 7.2점이지만, 
 당신의 예상 평점은 8.5점입니다.
 이유: 당신이 선호하는 비선형 서사 구조와 
       자주 높게 평가한 감독 크리스토퍼 놀란의 작품입니다."`

- 협업 필터링 + 콘텐츠 기반 필터링 결합
- LLM이 예측 근거를 자연어로 설명
- "이건 당신 취향 아닐 수 있어요" 같은 역추천도 가능

### **🎬 시청 여정 자동 기록 (Viewing Journey)**

`"2024년 1월, 우울한 시기에 주로 위로가 되는 
 휴먼 드라마를 찾아보셨네요.
 3월부터는 액션 영화 비중이 늘었습니다."`

- 시청 패턴에서 심리 상태, 관심사 변화 자동 감지
- LLM이 시청 이력을 스토리로 재구성
- "1년 전 봤던 이 영화, 지금 다시 보면 어떤 느낌일까요?" 제안

### **💬 리뷰 큐레이션 어시스턴트**

`사용자: "이 영화 연출 평가가 궁금해"
시스템: 300개 리뷰 중 연출 언급 리뷰만 추출 및 요약
       "70%가 카메라 워크를 극찬했으나, 
        25%는 편집이 산만하다고 지적했습니다."`

- RAG로 특정 관점 리뷰만 필터링
- LLM이 의견 분포와 대표 의견 요약
- 스포일러 자동 제거 버전 제공

### **🎭 캐릭터/배우 기반 추천**

`"히스 레저의 조커처럼 카리스마 있는 
 악역이 나오는 다른 영화 추천"`

- 배우의 다른 작품이 아닌, "유사한 캐릭터 특성" 기반 추천
- RAG로 캐릭터 분석 데이터 검색
- LLM이 캐릭터 간 유사성 판단

### **📺 에피소드 스킵 가이드 (Smart Episode Guide)**

`"시즌2 에피소드 4는 평가가 낮으니 건너뛰고 
 5화부터 보셔도 스토리 이해에 문제없습니다.
 4화 핵심 내용: [스포일러 없는 요약]"`

- 에피소드별 평점 + 리뷰 분석
- LLM이 건너뛰어도 되는 에피소드 판단
- 필수 정보만 요약 제공

### **🔄 재시청 가치 평가 (Rewatch Value Analyzer)**

`"이 영화는 반전을 아는 상태에서 다시 보면 
 새로운 복선들을 발견할 수 있습니다. 재시청 추천도: 상"`

- 리뷰에서 "다시 봤더니", "재관람" 관련 내용 추출
- 반전형 vs 분위기형 작품 구분
- 재시청 타이밍 제안 ("6개월 후 다시 보면 새로울 거예요")

### **🎨 영화 관계도 시각화 (Cinematic Universe Map)**

`"기생충을 좋아했다면:
 → 주제: 설국열차 (계급 갈등)
 → 연출: 마더 (미스터리 풀어가는 방식)
 → 분위기: 버닝 (불편한 긴장감)"`

- 단순 "비슷한 영화" 대신 "어떤 면에서 비슷한지" 구조화
- LLM이 영화 간 연결고리를 다각도로 분석
- 지식 그래프 형태로 탐색 가능

### **🎤 가상 평론가 모드**

`"이 영화를 봉준호 스타일로 평가한다면?"
"마틴 스코세이지가 이 영화를 보면 뭐라고 할까?"`

- LLM이 특정 감독/평론가 관점으로 영화 분석
- 다양한 시각에서 작품 이해
- 교육적 목적으로도 활용 가능

### **⚡ 실시간 감상 동기화 (Live Sync Commentary)**

`[함께 보기 중]
시스템: "방금 나온 장면, 3명이 모두 놀란 반응을 보이셨네요!"
        "이 대사에 대해 각자 다른 해석을 하고 계신 것 같은데,
         대화해보시겠어요?"`

- 여러 명이 동시 시청 시 반응 타이밍 자동 감지
- LLM이 공통 관심 포인트 찾아서 대화 촉진
- 시청 후 하이라이트 자동 요약

### **🎯 역추천 (Anti-Recommendation)**

`"이 영화는 평점은 높지만 당신 취향은 아닐 것 같습니다.
 이유: 느린 템포의 다큐멘터리 스타일인데, 
       당신은 주로 빠른 전개를 선호하십니다."`

- 높은 평점이라도 개인 취향과 안 맞으면 경고
- 시간 낭비 방지
- "그래도 한번 도전해볼까요?" 옵션 제공

### **📚 영화 교육 모드 (Film School Mode)**

`"이 장면의 미장센 분석:
 - 인물 배치: 주인공이 화면 하단에 위치해 압박감 표현
 - 조명: 측면 조명으로 얼굴 절반만 밝혀 이중성 암시
 - 색감: 차가운 블루 톤으로 고립감 강조"`

- RAG로 영화 이론, 기법 데이터베이스 구축
- LLM이 장면별 영화 문법 해설
- 입문자도 영화를 깊이 이해할 수 있도록 지원

### **🎯 감성 기반 검색 + 설명 가능한 추천**

**구현 난이도**: ⭐⭐⚫⚫⚫ (중하)
**차별화 효과**: ⭐⭐⭐⭐⭐
**예상 소요**: 1주

`사용자: "비 오는 날 보기 좋은 영화"
시스템: 
- 검색: "rainy day", "cozy", "melancholic" 키워드로 벡터 검색
- 추천: "어바웃 타임"
- 설명: "이 영화는 잔잔한 배경음악과 따뜻한 색감이 특징이며,
        당신이 높게 평가한 '로맨틱 코미디' 장르와 일치합니다."`

**구현 방법**:

**왜 이것부터?**

- 기존 서비스와 가장 명확한 차별점
- 사용자가 즉각 체감 가능
- RAG의 핵심 기능 직접 활용
- 추후 확장 가능 (취향 학습 → 더 정교한 추천)

### **📝 리뷰 큐레이션 & 자동 요약**

**구현 난이도**: ⭐⭐⚫⚫⚫ (중하)
**차별화 효과**: ⭐⭐⭐⭐⚫
**예상 소요**: 4-5일

`사용자: "이 영화 연출 평가 궁금해"
시스템:
[50개 리뷰 중 연출 관련 15개 추출]
긍정 (80%): "카메라 워크가 예술적", "롱테이크 압도적"
부정 (20%): "과도한 슬로우모션"
대표 리뷰: "박찬욱 감독의 색채 활용이 돋보인다..."`

**구현 방법**:

- 기존 리뷰 데이터 크롤링 (네이버 영화, IMDB 등)
- 키워드 추출 후 관련 리뷰 RAG로 검색
- LLM으로 요약 + 감정 분류 (긍정/부정)
- 스포일러 키워드 사전 만들어 자동 필터링

**왜 필요한가?**

- 리뷰가 많은 영화는 읽기 부담
- 특정 관점만 빠르게 파악 가능
- 구현 복잡도 낮음 (크롤링 + LLM 요약)

### **💬 간편 리뷰 자동 생성 (선택사항)**

**구현 난이도**: ⭐⭐⭐⚫⚫ (중)
**차별화 효과**: ⭐⭐⭐⚫⚫

시스템 생성:
"영상미와 연출은 훌륭했으나 결말이 다소 아쉬웠습니다. 
 전반적으로 만족스러운 작품이었어요."`

**구현 방법**:

- 사용자가 별점 + 간단한 태그만 선택
- Few-shot 프롬프트로 LLM이 리뷰 초안 작성
- 사용자가 수정 후 게시
- (시간 남으면) 사용자별 문체 학습

**왜 도움되는가?**

- 리뷰 작성 진입장벽 낮춤
- 더 많은 사용자 참여 유도
- 데이터 축적 효과

### **🎭 "이런 요소" 검색**

**구현 난이도**: ⭐⭐⭐⚫⚫ (중)
**차별화 효과**: ⭐⭐⭐⭐⚫
**예상 소요**: 3-4일

`사용자: "반전이 충격적인 스릴러"
시스템: 
- RAG로 리뷰에서 "반전", "충격", "예상 못함" 키워드 검색
- "샤터 아일랜드", "유주얼 서스펙츠" 추천
- 스포일러 없이 "관객 90%가 예측 실패했다고 평가" 제공`

**구현 방법**:

- 영화별로 특징 태그 자동 추출 (리뷰 분석)
- "반전", "요리 장면", "복선" 같은 서사 요소를 메타데이터화
- 자연어 쿼리 → 태그 매칭 → 검색

**왜 강력한가?**

- 장르만으로는 찾기 어려운 세밀한 검색
- 사용자가 "이런 게 보고 싶었어" 순간 제공

## 🎨 1. 무드 기반 영화 매칭 (Color Psychology)

### **컨셉**

색깔/이모지로 현재 기분을 표현하면 영화 추천

### **구현 방식**

`[사용자 인터페이스]
"지금 기분이 어때요?"

🔴 열정적이고 에너지 넘침
🔵 차분하고 사색적
🟡 밝고 경쾌한 기분
🟣 신비롭고 몽환적
⚫ 우울하고 진지한
🟠 따뜻하고 포근한`

### **LLM 활용**

python

`prompt = f"""
사용자가 선택한 무드: {selected_color} - {mood_description}
이 기분에 어울리는 영화 특성:
- 색감/톤: {color_palette}
- 템포: {pace}
- 감정 곡선: {emotion_curve}

추천 영화와 이유를 설명하세요.
"""
```

### **차별화 포인트**
- Taranify처럼 색채 심리학 차용하지만, 한 단계 더 깊게
- "빨강 = 액션"이 아니라 "빨강인데 지금 피곤함 = 가벼운 코미디 액션"
- LLM이 색상 + 시간대 + 날씨(API 연동) 조합해서 추천

**구현 난이도**: ⭐⭐ (2-3일)
```

```
## 🎭 3. 영화 성격 테스트 (MBTI 스타일)

### **컨셉**
"당신의 영화 취향을 캐릭터로 표현"

### **질문 예시**
```
Q1. 영화 볼 때 더 끌리는 건?
A) 예상 못 한 반전이 있는 스토리
B) 감정이 깊게 와닿는 캐릭터

Q2. 영화가 끝난 후 느낌은?
A) "와... 이게 무슨 의미지?" (해석 욕구)
B) "배우 연기 미쳤다..." (감정 여운)

... (7-8개 질문)
```

### **결과**
```
🎬 당신의 영화 취향 타입: "깊이 탐험가"

특징:
- 다층적 서사 구조를 즐김
- 열린 결말을 좋아함
- 비주류 감독 작품에 관심 많음

비슷한 타입: 크리스토퍼 놀란 팬, 데이비드 핀처 애호가

이 타입이 좋아할 영화:
- 메멘토, 프라이머, 인셉션 ...

[친구에게 공유하기]
[이 타입 사람들과 대화하기]`

### **LLM 활용**

python

`# 사용자 답변 분석
answers = ["A", "B", "A", "A", "B", ...]

prompt = f"""
사용자 답변 패턴: {answers}
이 사용자의 영화 취향을 분석하고,
4가지 축으로 성격을 분류하세요:

1. 서사 vs 감성
2. 대중성 vs 예술성
3. 빠른템포 vs 느린호흡
4. 명확한결말 vs 열린해석

그리고 재미있는 별명을 지어주세요.
"""
```

### **재미 요소**
- 결과를 SNS에 공유 (바이럴 효과)
- "당신과 97% 일치하는 사람" 매칭
- 매월 취향 변화 추적 (예: "이번 달은 액션을 더 봤네요!")

**구현 난이도**: ⭐⭐⭐ (3-4일)
```

## 왜 이 조합인가?

### **영화 룰렛**

- 구현 간단하지만 **즉각적 재미**
- "결정 피로" 해소 = 실제 pain point 해결
- 데모 시 임팩트 큼 ("10초 추천" 퍼포먼스)

### **영화 운세**

- **매일 오는 이유** 제공 (retention)
- SNS 공유 유도 ("오늘 내 영화 운 ㅋㅋ")
- LLM이 매일 다른 문구 생성 → 질리지 않음

### **무드 기반 추천**

- Taranify 레퍼런스 차용 → "우리도 심리학 썼어요" 어필
- 시각적으로 예쁨 (UI/UX 강점)
- 기능적 추천과 감성적 재미의 중간 지점

`# 영화 룰렛 핵심 로직
def movie_roulette(user_context):
    # 1. 후보 영화 10개 추출 (RAG)
    candidates = vector_search(user_context, limit=10)
    
    # 2. LLM에게 1개만 선택하라고 강제

    {candidates}

## 기능별 AWS 서비스 매핑

### **1. 감성 기반 검색 + 설명 가능한 추천**

### **영화 데이터 저장**

- **Amazon RDS PostgreSQL with pgvector**
    - 역할: 영화 메타데이터 + 벡터 임베딩 저장
    - 용량: db.t3.medium (2 vCPU, 4GB RAM) 충분
    - 비용: ~$60/월
    - 이유: PostGIS 이미 사용 예정 + pgvector 확장으로 벡터 검색 통합 가능

**또는**

- **Amazon OpenSearch Service**
    - 역할: 벡터 검색 전용 (k-NN 지원)
    - 용량: t3.small.search (1개 노드)
    - 비용: ~$30/월
    - 이유: 벡터 검색 성능 우수, 확장성 좋음
    - ⚠️ 주의: 학습 곡선 있음

**추천**: PostgreSQL + pgvector (이미 DB 사용 예정이므로 통합 관리 용이)

### **임베딩 생성**

- **Amazon Bedrock (Titan Embeddings V2)**

# Perplexity - 기존 기능의 한계와 문제점 / LLM, RAG 활용 방안

## 1. 기존 기능의 한계와 부족한 점

## 1-1. 추천 시스템 측면

- 추천 기준의 세밀함 부족
    - 장르·국가 수준의 거친 태그 중심이라 “음악, 템포, 분위기, 캐릭터 유형” 같은 섬세한 취향까지 반영하지 못함.
    - 사용자가 싫어하는 요소(잔혹함, 불륜 서사, 너무 긴 러닝타임 등)를 명시적으로 제외하기 어렵다.
- 피드백 루프 부족
    - “이 영화는 추천 좋았어/별로였어”에 대한 간단한 피드백만 수집하고, 그 피드백이 어떻게 다음 추천에 반영되는지 체감하기 어렵다.
    - 사용자가 직접 취향을 조정(슬라이더, 체크박스 등)하며 “추천 결과를 튜닝”할 수 있는 인터랙션이 부족하다.

## 1-2. 평점·리뷰 활용 측면

- 리뷰의 “구조화” 부재
    - 대부분 자유 형식의 텍스트라서, 스토리·연출·연기·음악·OST·세계관 같은 요소별로 평가를 구분해서 보기 어렵다.
    - “결말이 마음에 안 든 경우가 많은지”, “OST가 호평받는지” 같은 질문을 바로 알기 어렵다.
- 사용자 상태별 관점 부족
    - “평일 퇴근 후에 가볍게 보기 좋은지”, “집중해서 보기 좋은지” 같은 맥락이 평점/리뷰에서 분리되어 있지 않다.
    - 혼자 봤을 때와 친구/연인/가족과 봤을 때의 평가 차이 같은 정보도 드러나지 않는다.

## 1-3. 결정 과정(무엇을 볼지 고르기) 측면

- 디스커버리 vs. 결정의 분리 실패
    - 새 작품을 “발견”하게 만드는 기능은 많지만, 최종적으로 1개를 고르는 단계에서 도움을 주지 못한다.
    - 후보 리스트는 길게 보여주지만, “3개로 추려주고 선택을 돕는” 단계가 부족하다.
- 사용자 목표 기반 UX 부족
    - “오늘은 그냥 멍하게 보고 싶다”, “영감 얻고 싶다”, “연인과 의미 있는 대화를 나누고 싶다” 같은 목표를 기준으로 콘텐츠를 안내해주지 않는다.
    - 추천이 콘텐츠 중심이라, 사용자의 ‘오늘의 목적·상태’ 중심으로 재정렬되지 않는다.

## 1-4. 콘텐츠 이해·학습 측면

- 깊이 있는 해석/유도 기능 부족
    - 예술영화나 복잡한 서사에서, 작품의 상징·메시지·감독 의도를 이해하도록 안내해주는 기능이 거의 없다.
    - 엔딩 해석이나 다양한 해석 관점을 소개해주는 “사후 감상 가이드”도 부족하다.
- 접근성·이해도 차이 반영 부족
    - 영화에 익숙하지 않은 이용자(입문자), 특정 장르를 처음 접하는 사람을 위한 난이도 조절이 없다.
    - “입문자용 해설”, “매니아용 심화 해설”을 나누어 제공하지 않는다.

## 1-5. 함께 보기·커뮤니티 측면

- 관계 중심 기능 부족
    - 함께 보는 사람(연인·친구·가족) 간의 관계를 고려해 추천하거나, 감상 후 관계를 돌아보게 하는 디자인이 없다.
    - 감상을 공유한 뒤 서로의 차이를 비교·대화할 수 있도록 정리해 주는 기능이 부족하다.
- 콘텐츠-커뮤니티 연계 부족
    - 토론 게시판, 댓글은 있지만 “특정 장면”과 바로 연결된 토론 스레드 또는 클립 단위 토론이 약하다.
    - “이 장면에 대한 대표 찬반 논점”이 정리되어 있지 않아, 토론을 따라가기 어렵다.

## 1-6. 사용자 취향 데이터·자기 이해 측면

- 자기 인식 도구의 부재
    - 서비스들은 사용자의 데이터를 많이 모으지만, 사용자가 “내가 어떤 취향의 사람인지, 어떻게 변해 왔는지”를 직관적으로 이해하도록 돕지 않는다.
    - 시간에 따른 취향 변화(10대 때 vs 지금), 시즌별/상황별 취향 패턴을 보여주지 않는다.
- 서비스 간 이동성 부족 (락인)
    - 특정 서비스에서 쌓은 취향·리스트를 다른 서비스나 OTT로 자연스럽게 옮기기 어렵다.
    - “나의 취향 데이터”를 내가 소유하고 활용한다는 감각이 부족하다.

## 2. LLM/RAG로 추가할 수 있는 차별화 기능 아이디어 (PDF 하단 내용 확장/재구성)

## 2-1. 세밀한 취향 프로파일링 & 대화형 튜닝

- 기능 아이디어
    - LLM이 사용자의 과거 시청작, 평점, 리뷰를 분석해 “당신은 ‘서서히 쌓이는 감정선 + 현실적인 대사’를 좋아하고, ‘급작스러운 전개 전환’을 싫어합니다.” 같은 자연어 취향 리포트를 생성.
    - 대화형으로 “이런 요소는 더 강화해줘 / 줄여줘”라고 말하면 취향 프로필이 실시간 업데이트되는 인터랙티브 튜닝.
- 차별화 포인트
    - 기존 서비스의 단순 태그/장르 취향을 넘어, 서사 구조·감정선·연출 스타일 수준까지 세밀하게 반영.
    - 사용자가 능동적으로 자신의 추천 알고리즘을 ‘조립’하는 경험 제공.

## 2-2. 요소별 리뷰 구조화·스코어링

- 기능 아이디어
    - LLM이 많은 리뷰를 읽고, 스토리·연기·연출·OST·세계관·캐릭터 등 요소별로 긍/부정 의견을 자동 분류.
    - 각 요소별 요약과 점수(예: OST 4.6/5, 스토리 3.8/5)를 생성하여 카드 하나로 보여줌.
- 차별화 포인트
    - “스토리는 평범하지만 연기가 미쳤다”, “음악 때문에 볼 가치가 있다”를 한눈에 파악 가능.
    - 유저는 자신의 관점(연기/OST 중시 등)에 따라 작품을 더 잘 고를 수 있다.

## 2-3. 목표·상황 기반 “시나리오 추천”

- 기능 아이디어
    - 사용자가 자연어로 “오늘은 친구들이랑 술 마시면서 틀어놓고 반쯤만 볼 거야. 너무 진지하지 않았으면 좋겠어.”라고 입력하면, LLM이 이를 구조화된 조건으로 해석.
    - RAG로 해당 조건에 맞는 후보를 뽑고, LLM이 “이런 시나리오에 특히 잘 맞는 작품 Top 3”를 스토리 형식으로 설명.
- 차별화 포인트
    - 단순 필터가 아닌, “상황/목적 스토리”를 이해하는 추천.
    - OTT 홈의 정렬이 아니라, 사용자의 하루·상황을 중심에 둔 UX.

## 2-4. 감상 후 대화·회고를 돕는 “애프터케어 AI”

- 기능 아이디어
    - 영화 다 보고 나면, LLM이 “이 작품에서 기억에 남는 장면 3개를 이야기해줘” 같은 질문을 던지고, 사용자의 답변을 요약해 개인 감상 노트 생성.
    - “이 영화가 당신의 과거 취향과 어떻게 다른지”, “앞으로 뭘 더 좋아하게 될지”에 대한 회고형 코멘트 제공.
- 차별화 포인트
    - 단순 로그 기록이 아니라, 감상 경험을 정리·반추하게 도와 주는 저널링 경험.
    - 장기적으로 “영화가 나를 어떻게 바꿔 왔는지”를 보여주는 스토리 제공.

## 2-5. 장면·클립 단위 감상 보조 + 토론 허브

- 기능 아이디어
    - 특정 장면의 타임코드를 기반으로, LLM이 그 장면에 대한 팬 토론/리뷰/해석을 모아 장면별 Q&A 페이지 제공.
    - 유저가 “이 장면에서 왜 이런 대사가 나오는지 이해가 안 돼”라고 질문하면, RAG로 대본·인터뷰·비평을 검색해 해설 제공.
- 차별화 포인트
    - 작품 전체가 아닌 “장면 단위”로 이해/토론에 접근.
    - 마니아·라이트 유저 모두가 각자의 깊이에서 작품을 즐기기 쉽다.

## 2-6. 관계·그룹 중심 추천/요약

- 기능 아이디어
    - 그룹(연인, 가족, 친구 모임)을 등록하면, 그룹 구성원의 취향을 LLM이 합성하여 “우리 그룹 전용 취향 프로필” 생성.
    - 시청 후 그룹 채팅·코멘트를 LLM이 요약해, “우리 커플/우리 친구들의 감상 패턴” 리포트를 제공.
- 차별화 포인트
    - 개인이 아닌 “관계 단위”로 콘텐츠 경험을 관리.
    - 추억 회상, 기념일 추천, 관계 맞춤 영화 마라톤 같은 경험을 설계할 수 있다.

## 2-7. 취향의 시간적 변화 추적·시각화

- 기능 아이디어
    - LLM이 연도·연령·계절별 시청 기록을 분석해 “당신은 겨울마다 느린 미장센 영화 비중이 늘어납니다” 같은 인사이트 제공.
    - “10년 전의 나 vs 지금의 나” 취향 비교 리포트와, 그 사이에 있었던 전환점 영화들을 스토리로 묶어 보여줌.
- 차별화 포인트
    - 단순 ‘올해 통계’가 아니라, “취향의 성장 스토리”를 제공.
    - 유저가 자기 자신을 더 잘 이해하고, 새로운 장르·스타일에 도전하게 만드는 동기 부여.

## 2-8. 오픈 포맷 취향 프로필 & 외부 연계

- 기능 아이디어
    - LLM이 내부 취향 데이터를 표준화된 스키마(선호 태그, 설명 텍스트, 예시 작품 목록 등)로 정리한 “내 취향 패키지” 생성.
    - 이 패키지를 다른 OTT나 서비스에서 읽어서, 초기 추천을 빠르게 맞출 수 있도록 하는 API/파일 포맷 구상.
- 차별화 포인트
    - “데이터를 쌓아서 가두는 플랫폼”이 아니라, “취향 데이터를 사용자에게 돌려주고 다른 곳에서도 쓰게 하는 플랫폼”이라는 포지셔닝.
    - 향후 B2B/에코시스템 확장 가능성도 갖게 됨.

`text[영화 카드]
🎬 오펜하이머

3. 추천 카드에 한줄 텍스트로 표시`

**기술 스택**

`textLLM: GPT-4o-mini (저렴) / Llama3.2 (무료)
프론트: 추천 리스트에 reason_text 필드만 추가
백: 사용자 좋아요 영화 3개 → 추천 대상 영화 → LLM → 10자 결과`

**예상 비용**: 월 5만원 이하 (사용자 1천명 기준)

## 🔧 **1위: "한줄 추천 이유" 상세 아키텍처**

```python
사용자 요청 → API Gateway → Lambda → Bedrock LLM → DynamoDB → 프론트엔드
```

```python
핵심 서비스
├── Lambda (Node.js/Python)
│   ├── 사용자 좋아요 영화 3개 조회 (DynamoDB)
│   ├── Bedrock에 "공통점 1줄 설명" 요청
│   └── 결과 캐싱 (DynamoDB TTL 24시간)
├── Bedrock (Claude 3.5 Sonnet / Llama 3.2)
│   └── "영화A와 영화B 공통점 15자로 설명해줘"
├── DynamoDB (사용자별 추천 캐시)
│   └── Partition Key: user_id, Sort Key: movie_id
└── API Gateway (REST API)
    └── /recommend/{movie_id}/reason`
```

**비용 최적화**

```python
Lambda: 1백만 호출 무료 → 3주간 충분

DynamoDB: 25GB 무료 티어 충분
```

## ① 정서 기반 취향 모델링

### 기능

LLM이 사용자의 리뷰·코멘트·대화 기록을 분석해

- 선호 감정 톤
- 서사 구조 선호
- 캐릭터 유형 선호
- 결말 선호

를 추출

### 차별점

“액션 좋아함”이 아니라

“성장형 서사 + 여운 있는 결말 선호” 같은 수준

### GPT의 구현가능성

정리하면, **영화·OTT 추천 분야에서 “LLM으로 사용자 리뷰/코멘트/대화 같은 비정형 텍스트를 분석하는 연구는 최근 각광받고 있지만 아직 초기 단계이거나 제한적으로만 활용되고 있는 게 사실이야.** 아래 근거를 정리했어 👇

---

## ✅ 1) LLM을 완전한 추천 엔진으로 쓰는 연구는 아직 초기 단계

- 일부 연구는 **LLM 자체를 추천 생성에 활용**하는 가능성을 보였지만 아직 **일반적인 추천 모델처럼 널리 쓰이지는 않음**이라고 평가돼.
    
    → 예: LLM이 자연어 사용자 요청을 받아 **대화형 추천**을 해보는 시도 연구가 있으나,
    
    **일반적인 개인화·정확도 측면에서는 한계가 있다는 의견도 존재함.**
    

---

## ✅ 2) LLM을 기존 추천 시스템 보조/설명 생성 용도로 활용하는 연구는 존재함

- LLM은 **설명 가능성(explainability) 제공 분야에서 사용됨** → “왜 이 영화를 추천했는지” 같은 자연어 설명 생성.
    
    → 그러나 **추천 성능 향상 연구는 아직 많지 않음**(논문이 소수).
    

---

## ✅ 3) “리뷰 텍스트 자체를 감성분석/추천에 활용”하는 연구는 존재함 (LLM까지는 아님)

- 일부 최신 영화 추천 연구는 **사용자 리뷰 텍스트를 감성분석해 추천 정확도를 높이는 시도**를 보고함
    
    → 리뷰 기반 추천 정확도가 전통적 평점 기반보다 향상.
    
    → 과거 연구는 BERT 같은 언어모델을 활용했지만, **LLM (GPT-계열) 연구는 아직 대체적이지 않음.**
    

---

## 🚫 즉, 요즘 영화 추천 시스템들은…

### ❌ 대부분은 아직도

- 평점 기반 추천
- 협업 필터링/콘텐츠 기반 벡터 추천
- 일부 감성분석 기반 하이브리드 추천
    
    이 중심이야
    

### ❗ 그리고 지금까지의 LLM 활용은

✔ LLM **설명 생성/대화형 추천 연구는 있음**

✔ **LLM으로 리뷰/대화 텍스트를 취향 모델링까지 완전히 대체해 쓰는 상용 시스템은 드묾**

→ 즉 여러분이 제안한

> “LLM으로 리뷰·대화·코멘트를 분석해서 정서/서사/캐릭터까지 취향 프로파일로 만든다”
> 
> 
> 이건 **현업 상용 서비스에는 아직 거의 도입되지 않았거나 연구 중인 최첨단 영역**이라고 할 수 있어.
> 

---

## 📌 왜 아직 널리 쓰이지 않을까?

### ❶ 기존 추천은 구조화 데이터 중심

- 평점 / 시청이력 같은 **정형 데이터**가 주로 추천 모델 입력임

### ❷ 텍스트 분석은 복잡

- 리뷰·코멘트는 단일 점수보다 **맥락/스포일러/토픽** 같은 **심층 정보**가 필요함
- LLM은 이런 **정서/서사 분석**에 강하지만,
    
    ➤ 추천 정확도 향상/평가가 어렵고
    
    ➤ 학습/비용 부담이 큼
    
    ➤ 사용자 벡터로 정량화하는 부분이 아직 연구 중임

## 🎯 정리: 너의 해결 방안은 “최신 연구 트렌드 + 아직 상용화 안 된 혁신 영역”

✔ LLM으로 리뷰/대화/코멘트 텍스트를 심층 분석 →

정서/서사/캐릭터 기반 취향 표현 만들기

→ **기존 추천 시스템에서는 거의 안 하고 있는 접근**임

✔ 논문 리뷰에서도

- 일부는 **LLM을 추천 설명・대화형 상호작용에 사용**함

## 2-6. 관계·그룹 중심 추천/요약

- 기능 아이디어
    - 그룹(연인, 가족, 친구 모임)을 등록하면, 그룹 구성원의 취향을 LLM이 합성하여 “우리 그룹 전용 취향 프로필” 생성.
    - 시청 후 그룹 채팅·코멘트를 LLM이 요약해, “우리 커플/우리 친구들의 감상 패턴” 리포트를 제공.

    - 개인이 아닌 “관계 단위”로 콘텐츠 경험을 관리.
    - 추억 회상, 기념일 추천, 관계 맞춤 영화 마라톤 같은 경험을 설계할 수 있다.
