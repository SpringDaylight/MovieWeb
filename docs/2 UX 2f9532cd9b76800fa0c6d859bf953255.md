## 자동 추천 기능은??

그럼 자동으로 먼저 추천해주는 기능은 딱히 없는 건가?

아주 정확한 질문이야.

이거 안 짚고 가면 **“그럼 앱 켜면 뭐부터 하지?”**라는 UX 공백이 생겨.

결론부터 말하면 👇

> 있다 / 없다는 이분법이 아니라
> 
> 
> **“우리는 자동 추천을 ‘주인공’으로 둘 건지, ‘조력자’로 둘 건지”의 선택 문제야.**
> 
> 그리고 지금 설계에선 **‘조력자’로 두는 게 훨씬 맞다.**
> 

아래에서 이유 + 어떻게 넣으면 좋은지 정리해줄게.

---

## 1️⃣ 왜 전통적인 “자동 추천 홈”을 안 중심에 둔 걸까?

### 기존 OTT의 자동 추천 방식

- 홈 열자마자
    - “지금 뜨는 콘텐츠”
    - “당신을 위한 추천”
- 사용자는 **받아들이는 입장**

### 너희 서비스의 핵심 철학

- 추천 ❌
- **판단 보조 / 예측 ⭕**

즉,

> “이거 봐”가 아니라
> 
> 
> **“이거, 너한테 맞을까?”**
> 

👉 그래서 **무조건적인 자동 추천을 중심에 두지 않은 것**이야.

---

## 2️⃣ 하지만 ❗ 자동 추천이 아예 없으면 생기는 문제

네 질문이 정확히 이 포인트를 찔렀어.

### 자동 추천이 없을 때 UX 문제

- 앱 첫 진입 시 할 게 없어 보임
- “검색해야만 쓰는 앱”처럼 느껴짐
- 초반 이탈률 ↑

👉 그래서 **‘약한 자동 추천’은 반드시 필요**함.

---

## 3️⃣ 우리가 가져가야 할 자동 추천의 정체

### ❌ 기존 OTT식 자동 추천

- 무한 스크롤
- 근거 없음
- “또 이 장르야…”

### ✅ 너희 서비스에 맞는 자동 추천

> “결정을 돕는 자동 제안”
> 

---

## 4️⃣ 사용자 관점에서의 자동 추천 분류

### 🟦 ① 상황 기반 오늘의 제안 (Soft Auto)

**홈 상단에 1~3개만**

- “오늘 당신에게 맞을 가능성이 높은 영화”
- “최근 취향 기준 만족 확률 Top 3”

📌 특징

- 적음 (선택 부담 X)
- 확률 + 이유 표시
- 클릭하면 바로 **판단 화면**으로 이동

👉 **자동 추천 + 예측 서비스 연결**

---

### 🟩 ② 내가 자주 보는 맥락 기반 추천

LLM이 사용자의 패턴을 요약해서 제안

- “요즘 잔잔한 영화 많이 보셨어요”
- “최근엔 관계 중심 서사를 선호하시네요”

👉 **추천이라기보다 ‘관찰 결과 제시’**

---

### 🟨 ③ 그룹 상황 자동 제안

그룹이 등록돼 있다면:

- “이번 주말, ○○그룹 기준 만족 확률 높은 영화”
- “지난번 다 같이 본 영화랑 결 비슷한 작품”

👉 이건 **기존 OTT가 거의 안 하는 영역**

---

## 5️⃣ 중요한 선 긋기 (이거 진짜 중요)

### 우리가 안 하는 자동 추천

- ❌ 끝없는 리스트
- ❌ “인기 콘텐츠”
- ❌ 근거 없는 큐레이션

### 우리가 하는 자동 추천

- ⭕ 소량
- ⭕ 확률 + 이유
- ⭕ 클릭 시 ‘판단’으로 연결

> 자동 추천은 ‘입구’일 뿐,
서비스의 목적지는 항상 ‘판단 화면’
> 

---

## 6️⃣ UI로 보면 이렇게 자연스럽다

### 홈 화면 구조 예시

```
[ 오늘의 제안 ]
✔ 만족 확률81%
  → 왜? (한 줄 설명)

✔ 만족 확률74%
  → 왜?

[ 직접 찾아보기 ]
“지금 이런 영화 찾고 있어요…”
(자연어 입력)

```

👉 자동 추천은 있지만

👉 **항상 ‘내가 판단하는 느낌’을 유지**

---

## 7️⃣ 한 줄 결론 (설계 기준 문장)

> “우리 서비스는 자동 추천을 제공하지만,
> 
> 
> 그것은 사용자를 대신해 고르는 기능이 아니라
> 
> 사용자의 판단을 시작하게 만드는 트리거 역할이다.”
>

# 4️⃣ 사용자 관점 기준 기능 묶음 (UI용 정제)

이제 UI를 만들 수 있도록 **기능을 ‘화면 단위’로 묶어볼게**.

---

## 🟦 [홈] 지금 뭐 보고 싶어?

### 사용자 인식

> “나한테 맞는 영화 좀 골라줘”
> 

### 제공 기능

- 자연어 입력
    - “잔잔한데 여운 큰 영화”
    - “오늘 생각 많아지는 거”
- 빠른 선택 태그
    - 혼자 / 연인 / 친구
    - 가볍게 / 진지하게

📌 내부적으로는

→ A-5 → A-3 → A-4 파이프라인 실행

---

## 🟩 [영화 판단] 이 영화, 나랑 맞을까?

### 사용자 인식

> “이 영화 볼지 말지 판단해줘”
> 

### 제공 기능

- 영화 검색 or 선택
- 결과 카드:
    - 👍 좋아할 확률 (예: 78%)
    - 🧠 왜 그런지 설명
    - ⚠️ 나랑 안 맞을 수 있는 포인트

📌 내부적으로는

→ A-1 → A-2 → A-3 → A-4

👉 **이 화면이 서비스의 핵심**

---

## 🟨 [같이 보기] 우리 기준으로는 어때?

### 사용자 인식

> “여럿이 보는데, 다 괜찮을까?”
> 

### 제공 기능

- 그룹 생성 (연인 / 친구 / 가족)
- 각자 취향 요약
- 결과:
    - 전체 만족 확률
    - 누가 특히 좋아할지 / 아쉬울지

📌 내부적으로는

→ 개인 파이프라인 × N

→ A-6 그룹 합성

---

## 🟪 [감상 기록] 보고 나서 남기기

### 사용자 인식

> “이 경험을 기억하고 싶다”
> 

### 제공 기능

- 타임스탬프 코멘트
- 한 줄 소감
- 그룹 감상 요약

📌 내부적으로는

→ B 감상 이후 경험 관리

→ 다시 A-1로 피드백
